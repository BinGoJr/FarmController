# FarmController

FarmController — система управления микроклиматом, поливом и фотофиксацией для экспериментальной установки на базе Arduino Mega 2560.

## Описание

Код написан на C++ и разбит на модули. Используемые библиотеки: `RTClib`, `DHT sensor library`, `ModbusMaster`, `SensirionI2cScd4x`, `SD`, `SPI`, `Wire`.

Ключевые функции системы:

- **Управление освещением** — две независимые секции светодиодных лент с ШИМ-регулировкой яркости по трём каналам каждая. Яркость меняется по тактикам в зависимости от дня эксперимента и времени суток. Свет включается по расписанию (8:00 – 24:00).
- **Автоматический полив** — измерение влажности почвы 5 ёмкостными датчиками и Modbus-датчиком. При средней влажности ниже 60% включается помпа; полив идёт до достижения 80% или не более 10 минут.
- **Фотофиксация** — автоматическое управление камерой и дополнительным освещением для съёмки растений утром (перед включением света), в 16:00 и вечером (после выключения света). Во время фотосессии основной свет отключается.
- **Сбор данных** — чтение температуры/влажности воздуха (2×DHT11), CO₂/температуры/влажности (SCD40), давления и расхода воды, NPK, pH, EC, влажности и температуры почвы (Modbus RS485). Все данные сохраняются на SD-карту в файл `datalog.csv`.
- **Логика работы** — конечный автомат на уровне полива (IDLE → измерение → полив → сохранение) с интервалом измерений 3 часа. Освещение и фото обновляются каждые 10 мс в основном цикле.

## Структура проекта

```
src/
├── farmController.ino   # Точка входа, инициализация и главный цикл
├── Config.h             # Все пины и константы
├── LightController.h/cpp  # Низкоуровневое управление MOSFET и реле
├── LightScheduler.h/cpp   # Расписание яркости по дням и времени суток
├── PhotoManager.h/cpp     # Управление фотосессиями
├── SensorManager.h/cpp    # Чтение всех датчиков
├── DataLogger.h/cpp       # Запись данных на SD-карту
└── WateringSystem.h/cpp   # Автоматика полива
```

## Установка прошивки и подготовка среды

### Шаг 1. Установить Arduino IDE

Скачать и установить [Arduino IDE](https://www.arduino.cc/en/software) версии 2.3.8 или новее.

### Шаг 2. Установить необходимые библиотеки

В меню Arduino IDE: **Скетч → Подключить библиотеку → Управлять библиотеками...**

Установить следующие библиотеки:

- `RTClib` by Adafruit (≥ 2.1.0)
- `DHT sensor library` by Adafruit (≥ 1.4.4)
- `ModbusMaster` by Doc Walker (≥ 2.0.1)
- `Sensirion I2C SCD4x` by Sensirion (≥ 0.4.0)
- `SD` (встроенная, не требует установки)
- `SPI` (встроенная)
- `Wire` (встроенная)

### Шаг 3. Скачать репозиторий

Сохранить репозиторий локально (ZIP или `git clone`).

### Шаг 4. Открыть прошивку

В Arduino IDE открыть файл `farmController.ino` из папки `src/`.

### Шаг 5. Настроить параметры

Все основные настройки сосредоточены в `Config.h` и в исходных файлах модулей:

| Параметр | Где менять |
|---|---|
| Дата начала эксперимента | `farmController.ino`, переменная `experimentStart` |
| Пороги влажности | `Config.h` — `LOW_MOISTURE_THRESHOLD`, `HIGH_MOISTURE_THRESHOLD` |
| Время включения/выключения света | `Config.h` — `Light::ON_HOUR`, `Light::OFF_HOUR` и т.д. |
| Тактики яркости по дням и времени | `LightScheduler.cpp` — массивы `phasesSec1[]`, `phasesSec2[]` и структуры `tactic_*` |
| Расписание фотосессий | `PhotoManager.cpp` — массив `photoSchedules[]` |
| Калибровка ёмкостных датчиков | `Config.h` — массивы `calibA[]`, `calibB[]` |

### Шаг 6. Загрузить прошивку

Подключить Arduino Mega 2560 к компьютеру, выбрать **Инструменты → Плата → Arduino Mega 2560** и нужный COM-порт, затем нажать **Загрузка**.

## Дополнительный код для ПК

Для полноценной работы системы используются три Python-скрипта.

### `FarmLog.py` — сбор данных с Arduino

Читает данные по Serial-порту, разделяет строки с префиксом `DATA:` (сохраняет в `datalog.txt`) и служебные сообщения (сохраняет в `system.txt` с временной меткой).

Настройка: укажите COM-порт в переменной `PORT` и `BAUDRATE` (должен совпадать с прошивкой — 57600).

Запуск: `python scripts/FarmLog.py`

### `Pollux_save.py` — автоматическая фотосъёмка камерой Pollux

Ожидает появления камеры в сети (IP `10.0.0.1`), отправляет команду на съёмку, скачивает 5 снимков и сохраняет в папку `Pollux_Photos/`. Цикл повторяется при каждом включении камеры (реле управляется Arduino).

Настройка: укажите `CAMERA_IP` и `SAVE_FOLDER`.

Запуск: `python scripts/Pollux_save.py`

### `Esp32-cam.py` — сервер для приёма фото с ESP32-CAM

Запускает HTTP-сервер, принимает POST-запросы на `/upload`. Доступ разрешён только с IP из списка `ALLOWED_IPS`.

Настройка: укажите IP-адреса ваших ESP32-камер.

Запуск: `python scripts/Esp32-cam.py`

### Установка зависимостей Python

```
pip install -r scripts/requirements.txt
```

## Использование системы

### Шаг 1. Подготовка SD-карты

Отформатируйте SD-карту в FAT32. При первом запуске система создаст файл `datalog.csv` с заголовками автоматически.

### Шаг 2. Подача питания

Подключите питание к Arduino и исполнительным устройствам (помпа, светодиоды, датчики). Система начнёт работу автоматически.

### Шаг 3. Мониторинг

Подключитесь к Serial-порту (скорость 57600). В порт выводятся:

- текущий день эксперимента при старте,
- строки `DATA:…` с данными датчиков после каждого измерения,
- сообщения `Watering started` / `Watering stopped`,
- сообщения `Photo session started` / `Photo session ended`.

### Шаг 4. Анализ логов

Извлеките SD-карту и откройте `datalog.csv`. Столбцы:

```
DateTime, Temp1, Hum1, Temp2, Hum2, PressureMPa, FlowRate, TotalVolume,
Soil1..Soil5, N, P, K, pH, SoilMoistureModbus, SoilTemp, EC,
CO2, SCD40_Temp, SCD40_Hum
```

Значение `-999` в любом поле означает ошибку чтения соответствующего датчика.
