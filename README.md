# FarmController

FarmController - Данный проект представляет собой систему управления микроклиматом, поливом и фотофиксацией для экспериментальной установки на базе Arduino Mega 2560. 
## Описание
Код написан на языке C++ с использованием библиотек `RTClib`, `DHT sensor library`, `ModbusMaster`, `SD`, `SPI`, `Wire`. Ключевые функции системы:
- **Управление освещением**: две независимые секции светодиодных лент с ШИМ-регулировкой яркости. Яркость меняется по фазам в зависимости от дня эксперимента. Свет включается по расписанию (8:00 – 24:00).
- **Автоматический полив**: измерение влажности почвы 5 ёмкостными датчиками + Modbus датчиком влажности. При средней влажности ниже 60% включается помпа, полив идёт до достижения 75% или максимум 10 минут.
- **Фотофиксация**: автоматическое управление камерой и дополнительным освещением для съёмки растений в заданные моменты (утром, в 16:00, вечером). Во время фотосессии основной свет отключается.
- **Сбор данных**: чтение температуры/влажности (2×DHT11), давления воды, расхода воды, NPK, pH, EC, температуры почвы (Modbus RS485). Все данные сохраняются на SD-карту в файл `datalog.csv`.
- **Логика работы**: конечный автомат (IDLE → измерение датчиков → полив → сохранение) с интервалом измерений 3 часа.

### Установка прошивки и подготовка среды

#### Шаг 1. Установить Arduino IDE
Скачать и установить [Arduino IDE](https://www.arduino.cc/en/software") версии 2.3.8 или новее.

#### Шаг 2. Установить необходимые библиотеки
В меню Arduino IDE: **Скетч → Подключить библиотеку → Управлять библиотеками...**  
Установить следующие библиотеки (указаны минимальные версии):
- `RTClib` by Adafruit (≥ 2.1.0)
- `DHT sensor library` by Adafruit (≥ 1.4.4)
- `ModbusMaster` by Doc Walker (≥ 2.0.1)
- `SD` (встроенная, не требует установки)
- `SPI` (встроенная)

#### Шаг 3. Скачать репозиторий
Сохранить данный репозиторий локально (как ZIP или через `git clone`).

#### Шаг 4. Открыть прошивку
В Arduino IDE открыть файл `farmContoller.ino` из папки `src/`.

#### Шаг 5. Настроить параметры (при необходимости)
Отредактировать в скетче:
- Дату начала эксперимента (`initExperimentStart()`).
- Пороги влажности (`LOW_MOISTURE_THRESHOLD`, `HIGH_MOISTURE_THRESHOLD`).
- Время включения/выключения света (`LIGHT_ON_HOUR`, `LIGHT_OFF_HOUR`).
- Фазы яркости для секций 1 и 2 (`phasesSec1[]`, `phasesSec2[]`).
- Расписание фотосессий (`photoSchedules[]`).

#### Шаг 6. Загрузить прошивку
Подключить плату Arduino Mega 2560 к компьютеру, выбрать в меню **Инструменты → Плата → Arduino Mega 2560**, затем **Инструменты → Порт** (выбрать нужный COM-порт). Нажать кнопку **Загрузка**.

## Дополнительный код для ПК

Для полноценной работы системы используются три Python-скрипта:

### 1. `FarmLog.py` – сбор данных с Arduino
Скрипт читает данные по Serial-порту, разделяет строки с префиксом `DATA:` (сохраняет в `datalog.txt`) и служебные сообщения (сохраняет в `system.txt` с временной меткой).

**Настройка:**
- Укажите правильный COM-порт в переменной `PORT`.
- При необходимости измените `BAUDRATE` (должен совпадать со скоростью в прошивке – 57600).
- Запуск: `python scripts/FarmLog.py`

### 2. `Pollux_save.py` – автоматическая фотосъёмка камерой Pollux
Скрипт ожидает появления камеры в сети (IP `10.0.0.1`), отправляет команду на съёмку, скачивает 5 снимков и сохраняет их в папку `Pollux_Photos/`. Цикл повторяется при каждом включении камеры (реле управляется Arduino).

**Настройка:**
- Укажите правильный `CAMERA_IP` и `SAVE_FOLDER`.
- Запуск: `python scripts/Pollux_save.py`

### 3. `Esp32-cam.py` – сервер для приёма фото с ESP32-CAM
Запускает HTTP-сервер принимает POST-запросы на `/upload` с изображением в теле. Доступ разрешён только с IP-адресов из списка `ALLOWED_IPS`.

**Настройка:**
- Укажите разрешённые IP ваших ESP32-камер.
- Запуск: `python scripts/Esp32-cam.py`

### Настройка аппаратной части

#### Схема подключения (основные узлы)
| Компонент                          | Пин Arduino        | Примечание       |
| ---------------------------------- | ------------------ | ---------------- |
| MOSFET канал 1 (секция 1, канал 0) | 7                  | ШИМ              |
| MOSFET канал 2 (секция 1, канал 1) | 6                  | ШИМ              |
| MOSFET канал 3 (секция 2, канал 0) | 5                  | ШИМ              |
| MOSFET канал 4 (секция 2, канал 1) | 4                  | ШИМ              |
| Реле питания датчиков              | 11                 | HIGH             |
| Реле помпы                         | 10                 | HIGH             |
| Реле камеры                        | 9                  | LOW              |
| Реле лампы 1 (фото)                | 12                 | LOW              |
| Реле лампы 2 (фото)                | 25                 | LOW              |
| Датчик расхода (сигнал)            | 8                  | INPUT_PULLUP     |
| Датчик давления (аналог)           | A5                 | 0-5 В            |
| Ёмкостные датчики почвы            | A0, A1, A2, A3, A4 | аналоговые входы |
| SD-карта (CS)                      | 53                 | SPI              |
| RS485 (TX1, RX1)                   | 18, 19             | для Modbus       |

**Примечание:** перед первым запуском откалибруйте ёмкостные датчики почвы – задайте значения `soilAirValue[]` и `soilWaterValue[]` в коде.

### Использование системы

#### Шаг 1. Подготовка SD-карты
Отформатируйте SD-карту в FAT32. При первом запуске система сама создаст файл `datalog.csv` с заголовками.

#### Шаг 2. Подача питания
Подключите питание к Arduino и исполнительным устройствам (помпа, светодиоды, датчики). Система начнёт работу автоматически.

#### Шаг 3. Мониторинг
Подключитесь к Serial-порту (скорость 57600) для просмотра отладочных сообщений:
- Данные датчиков выводятся после каждого измерения.
- Состояния полива и фотосессий также дублируются в Serial.

#### Шаг 4. Анализ логов
Извлеките SD-карту и откройте файл `datalog.csv` или откройте файлы логов в указанных вами папках `SAVE_FOLDER` Столбцы:
`DateTime,Temp1,Hum1,Temp2,Hum2,PressureMPa,FlowRate,TotalVolume,Soil1...Soil5,N,P,K,pH,SoilMoistureModbus,SoilTemp,EC`
